Como comparar documentos utilizando tnicas de NLP ?

Responder essa pergunta pode ser importante para recomendaes de filmes no Netflix ou para integrao de dados a partir de descries de produtos semelhantes, por exemplo. E muitas outras coisas, claro.

Uma possvel resposta para esse problema utilizar tcnicas de vetorizao de texto, combinando tf-idf e similaridade de cossenos.

Term frequency-inverse document frequency (TF-IDF)

TF-IDF apenas uma forma de medir a importncia relativa dos tokens no texto, transformando um documento em uma lista de nmeros. Isso importante, pois para calcular a similaridade de cosseno, voc precisa de dois vetores de documento ou, simplesmente, vetorizar o texto.

Por outras palavras, tf-idf uma transformao que voc aplica a textos para obter dois vetores de valor real e, dessa forma, conseguir aplicar a mtrica de similaridade. Ele informa a importncia relativa de uma palavra ou token em um documento. Se uma string aparece muitas vezes em um documento e raras vezes em outro documento, ento ela deve receber um peso maior, pois caracteriza esse documento.

Matematicamente, temos:

Exemplo: Em um documento qualquer, a string coca-cola lata 350 ml aparece 5 vezes. Temos, ao todo, 20 documentos e o termo coca-cola lata 350 ml aparece tambm em 8 deles. Ento, o peso tf-idf dessa palavra na representao vetorial deste documento ser 5*log(20/8). Em geral, quanto maior o peso tf-idf, mais importante a palavra na caracterizao do documento. Por outro lado, se uma palavra aparece n vezes em um documento, mas tambm est presente em todos os outros documentos, ento o peso tf-idf para essa palavra em cada documento ser zero.

Em geral, quanto maior o peso tf-idf, mais importante a palavra na caracterizao do documento.

Similaridade de cosseno

Uma vez que j temos o texto vetorizado podemos aplicar a similaridade de cosseno, uma das mtricas de similaridade mais populares em NLP. Como o prprio nome j revela, a similaridade dada pela cosseno do ngulo entre os vetores.

Em lgebra linear, o prdouto vetorial de dois vetores igual ao mdulo de cada vetor multiplicado pelo cosseno do ngulo formado entre os vetores. Logo, cosseno do ngulo o produto escalar divido pela sua norma (vide imagem). A norma essencialmente o comprimento do vetor. Matematicamente, definido como a raiz quadrada da soma dos quadrados dos valores em todas as dimenses de um vetor.

Sejam dois vetories A e B, em que A:(4,7,1) e B:(5,2,3). Ento, o produto escalar dos vetores ser dado por e o produto da magnitude dos dois vetores ser dado por

Em NLP, os vetores de documento quase sempre utilizam pesos no negativos. Logo, o ngulo somente pode variar de 0 a 90 graus ou cos(0)=1 e cos(90)=0, onde 1 indica que os documentos so idnticos e zero que no h semelhana entre os documentos.

Em NLP, the cosine score varia entre 0 e 1, onde 0 indica nenhuma semelhana e 1 indica que os documentos so idnticos.

Implementao utilizando o scikit-learn

Uma maneira de aplicar a teoria atravs de um sistema de recomendao que sugere filmes com base na similaridade de enredos. Nossa tarefa construir uma funo que receba o ttulo de um filme e produza uma lista de filmes com enredos semelhantes. Por exemplo, se um usurio do Netflix um belo dia asssite um dos meus filmes favoritos High Plains Drifter (1973) de Clint Eastwood esperado que outros fimes sobre faroeste, bandidos ou mortes sejam sugeridos em seguida.

A funo de recomendao possui, basicamnete, 03 argumentos: o ttulo do filme, a matriz de similaridade de cosseno e uma srie de ndices. A funo extrai as pontuaes de semelhana de cosseno par a par de um filme com todos os outros filmes do dataset. Em seguida, ela classifica essas pontuaes em ordem decrescente. Finalmente, ela produz os ttulos dos filmes correspondentes s pontuaes mais altas de similaridade. Observe que a funo ignora a pontuao de similaridade mais alta (score = 1). Isso ocorre porque o filme mais semelhante a um determinado filme o prprio filme!

Uma dica substituir a funo cosine_similarity por linear_kernel. A sada permanece a mesma, mas leva muito menos tempo para calcular. A diferena no tempo de execuo entre as duas funes foi pequena (0.357 v. 0.339 seconds), porm quando se est trabalhando com grande volume de dados e seus vetores esto na representao tf-idf, uma boa prtica utilizar linear_kernel para obter um melhor desempenho.

import pandas as pd
import numpy as np
from sklearn.feature_extraction.text  import  TfidfVectorizer
from sklearn.metrics.pairwise import linear_kernel

# Importar dataset
metadata = pd.read_csv('data/movie_overviews.csv').dropna()
metadata.head()


indices = pd.Series(metadata.index, index=metadata['title']).drop_duplicates()

def get_recommendations(title, cosine_sim, indices):
    # Obter o ndice do filme que corresponde ao ttulo
    idx = indices[title]
    # Obter o score de similaridade
    sim_scores = list(enumerate(cosine_sim[idx]))
    # Classificar os filmes com base no score de similaridade
    sim_scores = sorted(sim_scores, key=lambda x: x[1], reverse=True)
    # Obter o score dos 10 filmes mais semelhantes
    sim_scores = sim_scores[1:11]
    # Obter os ndices de filmes
    movie_indices = [i[0] for i in sim_scores]
    # Retorna os 10 filmes mais parecidos
    return metadata['title'].iloc[movie_indices]


movie_plots = metadata['overview']


tfidf = TfidfVectorizer(stop_words='english')

# Construir a matriz TF-IDF
tfidf_matrix = tfidf.fit_transform(movie_plots)

# Gerar a matriz de similaridade de cosseno
cosine_sim = linear_kernel(tfidf_matrix, tfidf_matrix)

# Gerar recomendaes de filmes
print(get_recommendations("High Plains Drifter", cosine_sim, indices))
